Ambiente DX para Data

Um ambiente modular em PowerShell 7 para acelerar projetos de Data Science e Data Engineering com mais padronização, automação e reprodutibilidade.

PowerShell 7 Python 3.12 uv Ruff VS Code DX

Contexto

Em projetos de dados, uma parte importante do esforço costuma ser desperdiçada com problemas de ambiente: conflitos de versões, ativação manual de ambientes virtuais, ausência de padrão entre projetos e baixa previsibilidade operacional.

Este projeto foi desenvolvido para tratar o ambiente como parte da engenharia da solução, criando uma base reutilizável para novos projetos e para retomada rápida de projetos existentes.

Problema que o projeto resolve

Setup repetitivo

Cada novo projeto exigia recriar ambiente, estrutura e convenções.

Erros de contexto

Python errado, dependências inconsistentes e ativação manual de .venv.

Baixa reprodutibilidade

Sem uma base consistente, o projeto perde qualidade técnica e velocidade de evolução.

Solução proposta

A solução foi um ambiente modular baseado em PowerShell 7 + Python + uv, com responsabilidades distribuídas por módulos independentes: 

Microsoft.PowerShell_profile.ps1
        ↓
profile.d/
├── 00-env.ps1
├── 05-welcome.ps1

├── 70-bootstrap.ps1

Esse desenho permite isolar responsabilidades, facilitar manutenção e tornar o ambiente mais previsível.

Principais capacidades

  • Ativação automática de .venv ao entrar no projeto
  • Padronização do Python ativo por contexto
  • Integração moderna com uv
  • Diagnóstico rápido com comando doctor
  • Boilerplate inicial para novos projetos
  • Integração com VS Code
  • Suporte a lint e formatação com Ruff
  • Melhora da Developer Experience aplicada a dados

Fluxo de uso

Abrir PowerShell
→ entrar na pasta do projeto
→ autovenv ativa o ambiente
→ rodar doctor
→ desenvolver
→ executar scripts, notebooks ou pipelines
→ versionar

Impacto prático

O maior ganho deste projeto foi deslocar o esforço do setup para a solução analítica.

Antes Depois
Ativação manual de ambiente Autoativação por diretório
Baixa previsibilidade Diagnóstico rápido com doctor
Setup repetitivo Fluxo reaproveitável e padronizado
Mais tempo em infraestrutura local Mais foco em análise, modelagem e entrega

Stack tecnológica

PowerShell 7, Python 3.12, uv, VS Code, Ruff e estrutura modular via profile.d.

Conclusão

Este projeto mostra uma visão de engenharia aplicada ao trabalho com dados. Mais do que configurar terminal e ambiente virtual, a proposta foi criar uma base profissional, versionável e reutilizável para acelerar projetos de Data Science e Data Engineering.

Em resumo: um bom projeto de dados começa antes do notebook.
